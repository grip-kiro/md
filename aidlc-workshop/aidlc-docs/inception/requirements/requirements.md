# 테이블오더 서비스 요구사항 문서

## Intent Analysis Summary

- **사용자 요청**: 테이블오더 서비스 구축 (디지털 주문 시스템)
- **요청 유형**: New Project (신규 프로젝트)
- **범위 추정**: Multiple Components (프론트엔드 2개 + 백엔드 + DB)
- **복잡도 추정**: Moderate (다수 기능이 있으나 MVP 범위가 명확)
- **요구사항 깊이**: Standard

---

## 1. 기술 스택

| 구분 | 기술 |
|------|------|
| 프론트엔드 | React (TypeScript) |
| 백엔드 | Java + Spring Boot |
| 데이터베이스 | MySQL |
| DB 마이그레이션 | Flyway |
| 실시간 통신 | Server-Sent Events (SSE) |
| 인증 | JWT 토큰 기반 |

## 2. 아키텍처 구성

- **고객용 웹 앱**: 별도 React 애플리케이션 (테이블 태블릿용)
- **관리자용 웹 앱**: 별도 React 애플리케이션 (매장 운영자용)
- **백엔드 API 서버**: Spring Boot REST API (단일 서버)
- **데이터베이스**: MySQL
- **배포 환경**: MVP 단계에서는 로컬 개발 환경만 사용

## 3. 규모 및 성능 목표

- **대상 규모**: 중규모 (5~50개 매장, 매장당 20~50개 테이블)
- **MVP 단계**: 로컬 개발 환경에서 동작 확인

## 4. 기능 요구사항 (MVP 범위)

### 4.1 고객용 기능

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리

**사전 조건**:
- 관리자가 관리자 화면에서 테이블을 사전 등록 (테이블 번호, PIN 설정)
- 테이블 비밀번호: 단순 숫자 PIN (4~6자리)

**초기 설정 (관리자가 태블릿에서 1회 수행)**:
- 태블릿에서 마스터 PIN을 입력하여 설정 모드 진입
- 테이블 번호 및 테이블 PIN 입력
- 설정 완료 시 태블릿 전용 토큰 발급, 고객용 메뉴 화면으로 자동 전환

**JWT 토큰 전략**:
- Access Token(짧은 만료) + Refresh Token(긴 만료) 조합
- 태블릿용: Refresh Token 만료를 매우 길게 설정하여 사실상 상시 로그인 유지
- 관리자용: Access Token 짧은 만료 + Refresh Token으로 16시간 세션 관리
- 토큰 내 role 클레임으로 태블릿/관리자 구분 (role: "table" vs role: "admin")
- Access Token 만료 시 Refresh Token으로 자동 갱신

**토큰 저장**:
- 쿠키(Secure, HttpOnly, SameSite): JWT 토큰(Access + Refresh) 저장
- localStorage: 매장ID, 테이블번호, PIN 저장

**세션 종료 감지**:
- 관리자가 테이블 이용 완료 처리 시 태블릿 로그인 세션도 함께 종료
- SSE를 통해 서버에서 세션 종료 이벤트를 태블릿에 즉시 푸시 → 로그인 화면 전환
- Fallback: API 호출 시 401 응답 수신 시 로그인 화면 전환

**자동 재로그인**:
- 토큰이 무효화된 경우(서버 재시작, 이용 완료 등): localStorage에 저장된 정보로 자동 재로그인 시도
- 자동 재로그인 실패 시: 설정(로그인) 화면 표시

**기기 제한**:
- 하나의 테이블에 하나의 기기만 허용
- 새 기기 로그인 시 기존 세션 무효화

**네트워크 복구**:
- 네트워크 연결 끊김 후 복구 시 자동 세션 복구 시도 (토큰 유효하면 즉시 복구)

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시

**카테고리 탐색 UI**:
- 좌측 사이드바에 카테고리 목록 고정 표시
- 카테고리 선택 시 오른쪽 영역에 해당 카테고리 메뉴 목록 표시

**메뉴 카드 레이아웃**:
- 그리드 3열 레이아웃
- 각 카드: 이미지 + 메뉴명 + 가격
- 카드에 "+" 버튼 → 터치 시 수량 1로 바로 장바구니 추가 (수량 변경은 장바구니에서)
- 터치 친화적 버튼 크기 (최소 44x44px)

**메뉴 상세 정보**:
- 메뉴 카드 터치 시 모달/팝업으로 상세 정보 표시
- 모달 내용: 큰 이미지, 메뉴명, 설명, 가격, 수량 선택, 장바구니 추가 버튼

**품절 관리**:
- 관리자가 메뉴별 품절 상태 설정 가능
- 고객 화면에서 품절 표시 + 주문 불가 처리

**이미지 처리**:
- MVP 단계: 이미지 URL 직접 입력 방식
- 이미지 없는 메뉴: 기본 플레이스홀더 이미지(음식 아이콘) 표시

**데이터 캐싱**:
- 서버 측 캐싱 (Spring Cache) 적용
- 메뉴 변경(등록/수정/삭제/품절 변경) 시 캐시 무효화

**MVP 제외**: 메뉴 검색 기능 (카테고리 탐색만 제공)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 localStorage 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시 (5초) → 장바구니 비우기 → 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 매장 이용 완료 처리된 주문 제외

### 4.2 관리자용 기능

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증: Access Token(짧은 만료) + Refresh Token으로 16시간 세션 관리
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한: 계정 기반 (동일 계정 5회 실패 시 15분 잠금)
- 매장:관리자 관계: MVP에서는 1:1, 추후 1:N 확장 가능하도록 설계

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트 (2초 이내)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **테이블 사전 등록**: 관리자 화면에서 테이블 번호/PIN을 사전 등록 (태블릿 연결 전 필수)
- **마스터 PIN 관리**: 태블릿 설정 모드 진입용 마스터 PIN 설정
- **테이블 초기 설정**: 태블릿에서 마스터 PIN 입력 → 설정 모드 → 테이블 번호/PIN 입력 → 16시간 세션 생성
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- **테이블 세션 종료(이용 완료)**: 확인 팝업 → 주문 내역을 과거 이력으로 이동 → 현재 주문/총액 리셋
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리

**카테고리 관리**:
- 관리자가 카테고리 CRUD 가능 (추가/수정/삭제/순서 변경)

**메뉴 CRUD**:
- 메뉴 조회: 테이블(표) 형태 — 메뉴명, 카테고리, 가격, 품절 여부, 순서, 수정/삭제 버튼
- 메뉴 등록: 메뉴명, 가격, 설명, 카테고리(복수 선택 가능), 이미지 URL
- 메뉴 수정 / 삭제
- 메뉴 품절 상태 설정/해제

**메뉴-카테고리 관계**:
- N:M 관계 — 하나의 메뉴가 여러 카테고리에 속할 수 있음

**메뉴 노출 순서**:
- 드래그 앤 드롭으로 순서 변경

**메뉴 삭제 전략**:
- 소프트 삭제 — 메뉴를 비활성화(숨김) 처리, 기존 주문 내역에서는 메뉴명/가격 그대로 표시

**가격 데이터**:
- 정수(원 단위), 최소 0원, 최대 1,000,000원

**이미지 URL**:
- 검증 없음 — 관리자가 입력한 URL을 그대로 저장

**데이터 검증**:
- 필수 필드 검증 (메뉴명, 가격, 카테고리)
- 가격 범위 검증 (0 ~ 1,000,000)

## 5. 비기능 요구사항

### NFR-01: 이미지 관리
- MVP 단계: 이미지 URL 직접 입력 방식
- 추후: 파일 업로드 기능 추가 예정

### NFR-02: 데이터 관리
- Flyway를 통한 DB 스키마 버전 관리
- 주문 이력 데이터베이스 저장 (OrderHistory 테이블)
- 세션 ID로 주문 그룹화

### NFR-03: 세션 관리
- 테이블 세션: 첫 주문 시작 ~ 이용 완료 처리까지
- 관리자 세션: JWT 기반 16시간 유지
- 테이블 태블릿: 자동 로그인 (토큰=쿠키, 설정=localStorage)

## 6. 제외 기능 (구현하지 않음)

- 결제 처리 (카드, 현금, PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도/번역 API)
